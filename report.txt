____________________________________________________________________________
1. INTRODUCTION
____________________________________________________________________________
BusYatra is designed to address the fragmented nature of bus ticketing in Nepal. Currently, passengers must navigate multiple platforms such as Trip Turbo, Sajha Yatayat, and generic digital wallets, often facing real-time availability issues and a lack of personalized service. 

Objectives Recap:
- Provide an ACID-compliant mobile booking experience.
- Implement ML-based (KNN) seat recommendations for user personalization.
- Secure wallet-based transactions for reliable bookings.

Scope:
The system encompasses a cross-platform passenger mobile app, a comprehensive operator/admin web dashboard, and a scalable Node.js/Express backend.

3. METHODOLOGY AND IMPLEMENTATION STATUS
____________________________________________________________________________
3.1 Agile Methodology
The project followed an Agile Scrum approach. We have completed 2 main sprints (Approx. 4 weeks of development):
- Sprint 1: Foundation (Auth, DB Schema, Basic Routes) - DONE
- Sprint 2: Core Flow (Seat Layout, Booking Engine, Payment Integration) - DONE
- Sprint 3 (Ongoing): Personalization (ML Recommendation, Analytics Polish) - WIP

3.2 Requirements Status
Functional and Non-functional requirements are currently frozen. Minor adjustments were made to the payment verification flow to enhance reliability.

3.3 Algorithms Implementation Status
- 3.3.1 Seat Booking Integrity: Implemented via confirmed booking validation logic in `bookingController.js`.
- 3.3.2 ML-Based Seat Recommendation: Prototype phase. Data features (age, gender, history) are being collected via user profiles.
- 3.3.3 Search Optimization: Implemented. MongoDB indexes applied to `route`, `departureTime`, and `fare`.

3.4 Technology Stack (Current)
- Mobile: React Native + Expo (v51/52)
- Web: React + Vite + Tailwind CSS
- Backend: Node.js (v20+) + Express
- Database: MongoDB (Atlas)
- Documentation: Swagger / OpenAPI 3.0
